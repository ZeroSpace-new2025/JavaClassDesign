# AGENTS.md - java classs design 项目 AI 编码规范
OpenCode 必须严格遵守以下所有规则，所有代码生成、修改、重构均需符合本规范。

---

## 一、项目基础信息
- 技术栈 java21,swing
- 采用common->datamanager->business->visitable 四层结构
 - visitable 层只允许调用business 层
- 一个文件只有一个顶级类

---

## 二、编码规范（强制遵守）

### 1. 命名规范
- 符合java项目通用命名规范
- 常量：全大写+下划线，如 MAX_AMMO_COUNT
- 接口名：前缀 I，如 IEntity、IPoolable 

### 2. 注释规范
- 所有公共类、方法必须添加 javadoc 注释
- 复杂逻辑必须添加行内注释，说明设计意图


### 3. 需求来源自动读取需求（Demand.md）规范
- 必须完整解析对应项目/程序集/包 Demand.md 中所有人工编写的需求文本，无遗漏、无语义曲解
- 需求文本按最深的优先，与上层文本冲突时，需标注具体冲突点并等待人工确认
- 需求解析结果需输出结构化摘要，包含：核心目标、适用模块、约束条件、优先级
- 解析结果需与本规范（AGENTS.md）的编码规则对齐，确保需求落地符合项目编码要求
- 若需求文本存在歧义或冲突，需标记异常并输出明确的歧义点，待人工确认后再执行后续逻辑

### 4.自动编写程序集信息（AssemblyInfo.md）规范
- 严格遵循 AssemblyInfo.model.md 模板结构，自动生成 / 更新时需完整覆盖以下核心内容：
 - 程序集概述：包含编写环境信息、输入输出规范、依赖项说明
 - 程序集封装功能：明确类名及对应的功能描述，需与实际编码逻辑完全一致
 - 文件结构：准确填写类名及对应文件路径，路径需符合本规范「文件位置」章节要求
- 自动生成的程序集信息需通过格式校验：无语法错误、无信息缺失、无路径错误
- 程序集信息需与代码变更实时同步：代码修改（类名、功能、路径变更）后，需自动更新 AssemblyInfo.md 中对应条目
- 生成的 AssemblyInfo.md 内容需具备可读性，描述语言简洁、准确，符合游戏开发业务语境
- 禁止生成空值 / 默认占位符内容，所有字段需填充真实、有效的项目信息
